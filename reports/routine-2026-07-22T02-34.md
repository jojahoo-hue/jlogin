## What Remains / Next Ideas

**Expand the design library** — the biggest gap. Five requested motifs (metatron, mandala, harmonograph, spirograph, forest render) don't exist yet. Adding even two of these would significantly diversify the daily output.

**Mode-aware filenames for sacred designs** — currently `--mode silhouette` and `--mode outline` overwrite the same file. A simple fix in `sacred.py` / `cli.py` to suffix the filename (e.g. `sri-yantra-silhouette.svg`) would let both modes be committed in a single routine run.

**Cricut cut-path optimization** — the core stated purpose is SVG stencils for Cricut. None of the current generators add Cricut-specific attributes (cut paths, registration marks, layer separation). This would be high-value polish.

**Parametric variants** — the superformula and flower generators use fixed parameters. Exposing `--params` or named presets (e.g. `superformula --preset star`) would make the gallery more varied over time without adding new generators.

**Gallery viewer** — 130+ artefacts with no index. A simple HTML gallery page auto-generated by `python -m mathart.cli list --html` would make the output browsable.
