## What Remains / Next Ideas

### Core feature gaps
- **New sacred designs**: Metatron's Cube, Vesica Piscis, Seed of Life, Merkaba, Torus knot — all doable with existing `_svg.py` primitives.
- **New plot types**: Lissajous figures, spirographs (hypotrochoid/epitrochoid), reaction-diffusion patterns (Turing), wave interference.
- **Color / gradient SVGs**: current output is all-black stroke/fill; adding optional stroke palette or gradient fills would make exports more immediately usable for artistic work.
- **Multi-trace pipeline**: chain `render` → `trace` for several subjects (human figure, mandala photo, leaf), batch-generating a themed series.
- **Cricut export check**: SVG outputs haven't been verified in Cricut Design Space. A validation note or documented workaround for any incompatibility (e.g., `stroke-width`, nested groups) would complete the stated use-case.

### Quality / DX improvements
- **CI**: no GitHub Actions workflow exists yet. A simple `pytest` runner on push would catch regressions automatically.
- **Gallery index**: an auto-generated `gallery/README.md` with thumbnails (rendered via `cairosvg` or Playwright) would make the output browsable.
- **Parametric CLI flags**: expose `superformula` `m`, `n1`, `n2`, `n3` parameters via CLI so users can explore variants without editing Python.
- **Animated SVG**: SMIL or CSS animation on sacred geometry for web embedding.
