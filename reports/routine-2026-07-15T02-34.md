## What Remains / Next Ideas

### Priority: implement missing designs (referenced in spec for 3+ weeks)

The task spec has been referencing four designs that don't exist in the library.
Adding them would both satisfy the spec and meaningfully grow the generator catalogue:

1. **`metatron`** (Metatron's Cube) — 13 overlapping circles in the Fruit of Life
   arrangement. High-value sacred geometry target, natural complement to `flower`.
2. **`mandala`** — parametric with `--petals` and `--rings` flags. The spec mentions
   `--petals 12 --rings 4`; this API already exists in the CLI parser stub via `args.mode`
   pattern. Highest Cricut stencil value of the four.
3. **`harmonograph`** — Lissajous-family pendulum curves with decay (numpy only, no extra
   dep). Elegant output, straightforward to implement in `mathart/plot.py`.
4. **`spirograph`** — Hypotrochoid/epitrochoid (Roulette family). Very distinctive SVG
   output. Can share the `PLOTS` registry entry pattern.

### Other enhancements
- **`render` diversity**: only `horse` exists. Adding a tree or bird would diversify the
  trace pipeline and produce different Fourier contour shapes.
- **`sacred flower --mode silhouette`**: silhouette mode is currently only wired for
  `sri-yantra`. Extending it to `flower` would enable filled cuts of that design.
- **`--name` flag on sacred/plot**: both generators hardcode filenames. A `--name` override
  would allow generating the same design with different parameters without overwriting.
- **PNG output flag**: a `--png` option on trace (or a separate `rasterize` subcommand)
  would produce preview-ready rasters at target DPI without matplotlib hacks.
- **Batch `--all` flag**: regenerate every design in a single command, useful for
  after-parameter-change sweeps.
