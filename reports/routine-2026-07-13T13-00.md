## What Remains / Next Ideas

### What is finished and working

The `main` branch is significantly more mature than the `claude/prime-tDwAc` branch that was cloned locally. All the "next ideas" from previous reports are now implemented:

- Metatron's Cube, Mandala (--petals/--rings), Enneagram, Spiral, Seal, Wheel
- Harmonograph, Lissajous, Rose, Attractor, Phyllotaxis, Spirograph
- Forest and Turbines renders
- Stencil pipeline with marching squares, bridge generation, negative-mode SVG
- Fourier formula JSON serialization, stylize, kaleidoscope, morph, modulate
- Work validation (validate_work.inspect) catches black/flat renders automatically
- aigen (Gemini AI prompt to SVG) — dormant without API key

### What could come next

1. **`turbines-exact` full test coverage** — the `turbines-exact` render exists but is not explicitly tested in `test_all_registered_works_render_small` (it runs through REGISTRY, so it should be covered — worth verifying).

2. **`seal` design documentation** — the `seal` generator is listed but not described in the README; a docstring and test would help.

3. **Trace `--morph-with` gallery piece** — morph between horse and forest across t=0,0.25,0.5,0.75,1.0 would make a compelling animation-ready artefact set.

4. **Colorized SVG export** — current stencil pipeline is monochrome (cut-only). A `--color-bands N` flag on posterize mode could produce layered color SVGs for Print-then-Cut Cricut workflow.

5. **aigen integration test (mocked)** — the AI-gen path is tested for dormancy but not for actual output shape. A mock-key fixture test would protect the pipeline from silent regressions.

6. **NFT export pipeline** — Njaho's NFT project could benefit from a `mathart export-nft` command that bundles the SVG + formula JSON + PNG preview into a metadata-ready folder structure.
